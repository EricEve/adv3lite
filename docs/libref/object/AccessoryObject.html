Details

The AccessoryObject role is for a *third* object, beyond the direct and indirect objects, involved in the command. This might be a peer to the indirect object in an exchange (TRADE dobj AN iobj FOR AN accessory), but the canonical IF use is as a tool in a two-object operation (PUT dobj IN iobj WITH accessory, WRITE dobj ON iobj WITH accessory).
